Summary

1) 엔터프라이즈 애플리케이션의 가장 앞단에서 사용자 또는 클라이언트 시스템과 연동하는 책임을 맡고 있는 것이
  바로 웹 프레젠테이션 계층이다.
-> 자바의 데이터 액세스 기술도 종류가 많고, 다양하지만 웹 기술의 다양성에는 비할 수가 없다.
-> 자바의 웹 기술은 그 종류와 기술, 관련 프레임워크도 다양할 뿐만 아니라,
   각 기술이 배타적으로 사용되지 않고 여러 가지 방식으로 조합될 수 있으니
   최종적인 선택의 폭은 정말 넓다.
-> 좋게 말해서 선택의 폭이 넓은 것이고, 사실은 선택의 고민과 부담이 크다.
-> 게다가 데이터 액세스 기술과는 달리 웹 기술은 끊임없이 새로운 것이 등장하고 기존 기술도 빠르게 발전한다.


2) 스프링은 기본적으로 이렇게 기술의 변화가 잦은 웹 계층과 여타 계층을 깔끔하게 분리해서 개발하는 아키텍처 모델을 지지한다.
-> 그 덕분에 스프링 애플리케이션이지만 웹 계층은 어떤 기술로 대체하더라도 아무런 문제가 없게 만들 수 있다.
-> 물론 스프링 자신도 최고의 웹 기술과 웹 프레임워크를 제공하고 있다.


Insight
- 스프링의 웹 프레젠테이션 계층에서는 다양한 기술이 제공됨
-> 이에 대해 스프링 사용자는 여러 기술 중에서 선택을 할 수 있음

Application
- 우선은 스프링 MVC 중에서 스프링 서블릿에 대한 이해와 적용에 집중하자.
