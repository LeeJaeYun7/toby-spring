Summary

1) 스프링 부트의 역사
- 2012년, 스프링 프레임워크의 기능을 개선해줬으면 좋겠다고 올린 글에서 출발함
-> 'Containerless 웹 애플리케이션 아키텍처를 위해서 개선해줬으면 좋겠다'

- 스프링은 POJO라는 단순한 자바 오브젝를 이용해서 단순하게 개발이 가능한 프로그래밍 모델을 갖고 있음
-> 이걸 전통적인 자바 웹 환경에 적용하려면 알아야 할 지식들이 많이 있음
-> 더 단순한 방식으로 기본 지식 없이 개발을 빠르게 시작할 수 있고, 편리하게 사용할 수 있는 면에서 뒤쳐져 있음
-> 개선이 요구됨
-> 이러한 요구사항을 수용하기 위해서 새로운 프로젝트를 시작하자

- 2013년에 스프링 부트 0.5버전이 최초 공개됨
-> 스프링을 빠르고 간편하게 개발하면서 스프링의 가치를 지킬 수 있음
-> 2022년 10월에 2.7.5 버전이 공개됨

- 스프링 부트가 Containerless 웹 애플리케이션 아키텍처를 지원해달라는 요청에서 시작됨
-> Containerless 웹 애플리케이션 아키텍처란?


Insight
- 스프링 부트의 시작은 'Containerless 웹 애플리케이션 아키텍처'에 대한 요구에서 시작했다

Application
- 스프링과 스프링 부트는 아키텍처 관점에서 어떤 차이가 있는가?
-> 이에 대해 질문을 하고 이해를 할 필요가 있다.
